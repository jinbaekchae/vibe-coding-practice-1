# Business Requirements Document (BRD)

## Woka 출장 경비 정산 시스템

---

### 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | Woka 출장 경비 정산 시스템 |
| 배포 URL | https://woka-settle.vercel.app |
| 버전 | 1.0.0 |
| 상태 | 완성 |

### 2. 비즈니스 배경

Woka 조직의 출장 경비 정산 프로세스에서 다음과 같은 문제가 존재했다:

- **수기 입력 오류**: 직원이 유류비/통행료를 수동으로 입력하면서 발생하는 금액 오류
- **정산 지연**: 영수증 확인, 금액 검증, 수기 기록에 소요되는 시간
- **표준화 부재**: 정산 양식과 절차가 통일되지 않아 관리 부담 증가

### 3. 비즈니스 목표

| # | 목표 | 측정 지표 |
|---|------|-----------|
| 1 | 정산 소요 시간 단축 | 건당 처리 시간 |
| 2 | 금액 입력 오류 최소화 | AI 자동 추출 정확도 |
| 3 | 정산 데이터 중앙 관리 | Google Sheets 자동 기록률 |

### 4. 이해관계자

| 역할 | 설명 |
|------|------|
| 출장자 (사용자) | 출장 후 경비를 정산 신청하는 직원 |
| 경영지원팀 (관리자) | Google Sheets에서 정산 데이터를 확인/승인하는 담당자 |

### 5. 비즈니스 요구사항

#### BR-1. 유류비 자동 추출
- 사용자가 네이버지도/카카오맵 캡처 이미지를 업로드하면 AI가 "연료비" 금액을 자동 추출한다.
- 다크모드/라이트모드, 다양한 레이아웃을 지원해야 한다.
- 택시비, 통행료 등 다른 항목과 혼동하지 않아야 한다.

#### BR-2. 통행료 자동 추출
- 사용자가 카드사 앱의 고속도로 통행료 결제내역 캡처를 업로드하면 AI가 금액을 자동 추출한다.
- 추출된 통행료는 **5,000원 단위로 올림** 처리한다 (사내 정산 규정).

#### BR-3. 금액 수정 기능
- AI가 추출한 금액이 부정확할 경우 사용자가 직접 수정할 수 있어야 한다.
- 수정은 최종 확인 단계에서 이루어진다.

#### BR-4. 정산 데이터 자동 기록
- 최종 확인된 정산 데이터(이름, 강의명, 출장일, 유류비, 통행료)가 Google Sheets에 자동 기록된다.
- 별도의 백엔드 서버 없이 Google Forms API를 활용한다.

#### BR-5. 모바일 우선 접근성
- 출장 현장에서 스마트폰으로 즉시 정산할 수 있어야 한다.
- iOS/Android 모바일 브라우저에서 완전히 동작해야 한다.

### 6. 범위

#### 포함 (In Scope)
- 유류비 영수증 이미지 분석 (네이버지도/카카오맵)
- 통행료 결제내역 이미지 분석 (카드사 앱)
- 정산 신청 → 확인 → 완료 3단계 워크플로우
- Google Sheets 연동 데이터 저장
- 모바일 반응형 UI

#### 미포함 (Out of Scope)
- 사용자 인증/로그인
- 관리자 승인 워크플로우
- 정산 내역 조회/수정
- 알림 (이메일, 슬랙 등)

### 7. 제약 조건

| 제약 | 설명 |
|------|------|
| Vercel Body 제한 | API 요청 본문 최대 4.5MB → 이미지 리사이즈 필수 |
| Edge Runtime 타임아웃 | API 처리 최대 30초 |
| Claude API 의존성 | 이미지 분석 정확도가 AI 모델 성능에 의존 |
| Google Forms 제약 | 데이터 구조 변경 시 Form 재설정 필요 |

### 8. 성공 기준

- 유류비/통행료 AI 추출 정확도 95% 이상
- 모바일 브라우저에서 전체 플로우 정상 동작
- 정산 신청부터 완료까지 1분 이내 처리
